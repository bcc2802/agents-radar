## OpenClaw 项目动态日报 - 2026-06-20

### 1. 今日速览

今日 OpenClaw 项目异常活跃，社区参与度极高。24小时内产生了惊人的500个Issue和500个PR，尽管其中包含了大量由自动化机器人（如`clawsweeper`）管理的流程记录，但核心社区反馈与开发工作的密度依然很高。项目发布了 v2026.6.9-beta.1 测试版，并持续推进了多项关键修复。然而，潜流中涌现出多个P0/P1级别的严重内存泄漏、数据丢失和会话阻塞问题，项目健康度呈现“开发活跃、稳定性承压”的态势。新的 P0 内存泄漏 Bug（#91588）是目前最值得关注的单个风险点。

### 2. 版本发布

- **v2026.6.9-beta.1**: 该版本侧重于改善 Telegram 渠道的消息投递体验。更新内容包括：
    - Telegram 现在支持发送富文本（Rich HTML）。
    - 改进了 Markdown 和贴纸路径的保留与渲染。
    - 更准确地呈现进度草稿和命令输出。
    - 修复了提及（Mentions）和后台处理器（spooled handlers）在投递路径上的问题。

### 3. 项目进展

今日项目持续推进，主要进展体现在针对自动化流程和特定 Bug 的修复：

- **ClawSweeper 自动化合并**：多个 PR 已通过自动化流程进入待合并状态，包括清理孤立Cron会话（#84794）、加速命令启动（#89628）、修复 `doctor` 命令对正常Cron任务的不当建议（#94784）等，体现了项目在工具链本身维护上的投入。
- **核心功能修复**：在代码仓库中，有多个针对 Feishu 飞书渠道、Claude ACP 集成（#80499）、Windows 环境下的路径处理（#95212）的重要 PR 获得了标签更新或进入了审核流程。
- **社区贡献活跃**：新提交的 PR 包括对 TUI Markdown 渲染器（#95211）、子代理 `subagent_ended` Hook 文档（#95206/#95209）、以及 `openrouter` 提供商的模型 ID 处理（#95208）等问题的修复，显示社区正在积极贡献。

### 4. 社区热点

今日讨论最激烈的 Issue 集中在严重影响用户体验和系统稳定性的问题上：

1.  **Critical: Gateway Memory Leak (#91588) - 13条评论，1个👍**
    - **链接**: [Issue #91588](https://github.com/openclaw/openclaw/issues/91588)
    - **诉求**: 用户报告Gateway进程存在严重内存泄漏，RSS内存从350MB在2-3天内增长至15.5GB，最终导致OOM（内存溢出）崩溃和重启循环。这直接威胁到服务的长期稳定运行，是当前第一优先级的问题。

2.  **Per-agent memory-wiki vault configuration (#63829) - 10条评论，9个👍**
    - **链接**: [Issue #63829](https://github.com/openclaw/openclaw/issues/63829)
    - **诉求**: 这是一个高赞的功能需求。用户希望在多Agent架构中，每个Agent可以拥有独立的“记忆维基”知识库，而不是共享一个全局库。这表明社区对Agent个性化和数据隔离有强烈需求。

3.  **Subagent MCP工具注入失败 (#85030) - 8条评论，3个👍**
    - **链接**: [Issue #85030](https://github.com/openclaw/openclaw/issues/85030)
    - **诉求**: 用户反馈MCP（Model Context Protocol）工具无法注入到子Agent会话中。这严重阻碍了通过MCP扩展子Agent能力的用例，社区对此高度关注。

### 5. Bug 与稳定性

当前项目面临严重的稳定性挑战，多个高优 Bug 亟待解决。

- **P0 (Critical)**:
    - **[内存泄漏] Gateway内存泄漏 (#91588)**: 如上所述，RSS增长至15.5GB导致OOM，是当前最严重的稳定性 Bug。
    - **[数据丢失] memory-core异常删除每日记忆文件 (#84882)**: 记忆核心的“Dreaming”步骤会静默删除用户的日常记忆文件，属于数据丢失的高危bug。已有开放式PR关联。

- **P1 (High)**:
    - **[会话阻塞] 单一Agent会话阻塞整个Gateway事件循环 (#84903)**: 一个Agent的模型调用挂起会阻塞所有其他会话，破坏了会话隔离性。
    - **[性能回归] `openclaw doctor --fix` 4-5倍性能下降 (#85333)**: 诊断命令执行时间从55秒飙升到229秒以上，严重影响用户日常维护体验。
    - **[模型回退失效] 提供商级配额耗尽时模型回退链未触发 (#85103)**: 当模型提供商达到配额限制时，配置的回退机制不生效，导致服务中断。
    - **[邮件投递失败] 孤立Cron任务投递失败 (#92460, #91363)**: 多个Issue报告Isolated类型的Cron任务在执行后，其完成通知无法成功投递给用户，导致任务结果丢失。
    - **[配置错误] 会话错误选择authProfileOverride (#85126)**: 新创建的会话错误地选择了错误的认证配置，这可能导致API调用失败。

- **P2 (Medium)**:
    - **[安全] `web_fetch`忽略`NO_PROXY` (#93807)**: 可能导致内部服务请求被错误地路由到外部代理，存在安全隐患。
    - **[安全] Z.ai模型解析错误 (#94269)**: 静态目录中的Z.ai模型缺少`baseUrl`，导致请求错误地发往OpenAI，这既是功能Bug也是安全边界问题。

### 6. 功能请求与路线图信号

除了高优先级的 Bug 修复，社区也提出了一些有潜力的功能需求：

- **Agent 级记忆隔离 (#63829)**: 高赞需求，要求为多Agent场景提供独立的记忆知识库。这与提升多Agent能力的路线图高度吻合。
- **按频道/群组/私聊的模型覆盖 (#53638)**: 允许在不同对话场景下使用不同模型，而无需手动切换。这能极大提升配置灵活性和用户体验，预计可能会在后续版本中通过配置增强来实现。
- **主题会话家族 (#90916)**: 允许一个Agent拥有多个带有独立上下文的“主题”会话，同时共享持久记忆。这是一个更深层次的多语境管理功能，可能作为中期规划的一部分。
- **WebChat内联按钮支持 (#46656)**: 请求在WebChat UI中支持按钮交互。该需求已存在数月，且有相关PR，表明社区对WebChat交互性的提升有持续期待。

### 7. 用户反馈摘要

从Issue评论中可以提炼出以下真实用户痛点：

- **对稳定性极度的不安**：用户对于Gateway OOM崩溃（#91588）和会话阻塞（#84903）感到非常焦虑，因为这会直接导致服务不可用，影响生产环境。
- **对升级后果的担忧**：有用户反映升级到某些版本后（如v2026.6.8）导致了Telegram Web消息格式不兼容（#93794）或 `/usage` 功能失效（#93905），这表明用户对版本升级可能引入的回归问题非常敏感。
- **配置复杂且易错**：用户抱怨模型回退链（#85103）、认证配置选择（#85126）和子Agent工具注入（#85030）等功能的配置逻辑复杂且容易出错，希望有更直观和鲁棒的设计。
- **对诊断工具的依赖与失望**：`doctor --fix` 命令是用户遇到问题时的重要依赖，但其性能严重退化（#85333）甚至本身会引入新的配置问题（#85334），这极大地损害了用户的信任感。

2.  **跨模型智能已成为核心痛点**: 多个项目都暴露了在应对不同模型（特别是本地模型和国产模型）时的适配性问题。这意味着**构建一个稳健、可配置、能智能处理的模型抽象层和回退链**，将是未来框架的核心竞争力。
3.  **从“对话工具”到“协同平台”的转变**: **IronClab** 的SaaS特性、**NanoClaw**的权限继承、**ZeroClaw**的审批流程，以及社区的普遍诉求，都指向了一个趋势：AI Agent正在从一个简单的单用户对话接口，演进为**支持多用户、多Agent、有权限管理、可审计、可协作的团队工作平台**。
4.  **性能瓶颈的精细化管理**: **NanoBot** 报告了对Token编码性能瓶颈的优化，**CoPaw** 报告了向量库无限膨胀的问题。这表明，随着Agent长期运行和记忆功能的普及，**资源消耗的精细化管理**（如token优化、向量库维护、临时文件清理）将成为项目稳定性的区分点。

**对AI智能体开发者的参考价值**: 本报告揭示了一个事实：构建一个稳定、可靠、安全的个人AI智能体面临着多重挑战。建议开发者：
-   **首选成熟稳定的框架**：优先选择处于“质量巩固阶段”的项目作为基础，或对“快速迭代阶段”的框架进行充分的稳定性测试。
-   **将安全性作为第一优先级**：从设计之初就考虑认证、授权和输入过滤，避免依赖框架的默认配置。
-   **预留模型适配层**：设计插件化或可配置的模型路由与回退机制，以适应快速变化的模型生态。
-   **规划资源管理策略**：为长期运行的Agent设计合理的上下文管理、记忆清理和Token优化策略，防止因资源耗尽导致的崩溃。
